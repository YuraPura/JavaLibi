Модификаторы доступа в Java

Определяют видимость классов, методов и полей. Главная цель — инкапсуляция.

Основные модификаторы доступа

Модификатор Где видно
publicВезде (все классы, все пакеты)
protectedТот же класс + наследники + тот же пакет
default (нет слова)Только внутри того же пакета
privateТолько внутри того же класса

1. public

public class Car {
    public String model;           // видно везде
    public void start() { ... }    // видно везде
}

2. protected

Доступен в том же пакете и в наследниках (даже в других пакетах).

protected String vin;           // видно наследникам
    protected void checkEngine() { ... }

3. default (package-private)

Пишется без модификатора.

class Logger {                     // видно только в пакете
    void log(String msg) { ... }   // тоже только в пакете
}

4. private

Только внутри текущего класса — идеально для инкапсуляции.

public class BankAccount {
    private double balance;        // никто снаружи не увидит

    public void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    public double getBalance() {
        return balance;
    }
}

Золотое правило

Всегда начинай с самого строгого доступа (private) и открывай только то, что действительно нужно:
private → protected → default → public

Типичные ошибки

  • Публичные поля (кроме public static final)
  • Забывать про private у полей класса
  • Делать всё public «на всякий случай»
