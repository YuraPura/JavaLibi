Коллекции в Java: ArrayList и LinkedList

Рассмотрим две популярные реализации интерфейса List из Java Collections Framework.

1. ArrayList

ArrayList — динамический массив. Быстрый доступ по индексу.

Особенности:

  • Динамический размер — автоматически расширяется
  • Быстрый доступ по индексу (O(1))
  • Медленная вставка/удаление в середине (нужно сдвигать элементы)

Пример использования:

import java.util.ArrayList;

ArrayList numbers = new ArrayList<>();
numbers.add(5);
numbers.add(6);
numbers.add(2, 30); // вставка 30 на позицию 2

for (Integer el : numbers) {
    System.out.println(el);
}

// numbers.clear(); // очистка коллекции

2. LinkedList

LinkedList — двусвязный список. Быстрые вставки и удаления.

Особенности:

  • Каждый элемент хранит ссылки на предыдущий и следующий
  • Быстрая вставка/удаление в любом месте
  • Медленный доступ по индексу (нужно идти по цепочке)

Пример использования:

import java.util.LinkedList;

LinkedList numbers = new LinkedList<>();
numbers.add(10);
numbers.addFirst(5);
numbers.addLast(20);
// Методы те же, что и у ArrayList

3. Когда что выбирать?

Сценарий Лучше использовать
Частый доступ по индексу, мало вставок/удаленийArrayList
Частые вставки/удаления в серединеLinkedList
Нужны операции с концами списка (очередь, стек)LinkedList
Обычная работа со списком (90% случаев)ArrayList
На практике в 95% случаев выбирают ArrayList — он быстрее и проще.
